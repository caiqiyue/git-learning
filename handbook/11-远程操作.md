# 11. 远程操作：clone / fetch / pull / push

## 本节目标

学完这节后，你会：

## 本地和远程的关系

```
本地仓库                                    远程仓库 (GitHub)
┌─────────────────────┐                  ┌─────────────────────┐
│ .git/                │                  │                     │
│   refs/heads/main ──────── push ──────────→ refs/heads/main    │
│   refs/heads/dev ──────── push ──────────→ refs/heads/dev    │
│   refs/remotes/              ←── fetch ──── refs/heads/        │
│     origin/main            ←── pull ────── refs/heads/        │
└─────────────────────┘                  └─────────────────────┘
```

- **push**：把本地分支推送到远程
- **fetch**：把远程分支的元数据拉取到本地（不合并）
- **pull**：fetch + merge（拉取并合并）
- **clone**：把远程仓库完整复制到本地

## git clone

### 基本用法

```bash
# 克隆整个仓库
git clone https://github.com/username/repo-name.git

# 指定文件夹名称
git clone https://github.com/username/repo-name.git my-folder

# 克隆指定分支
git clone -b branch-name https://github.com/username/repo-name.git

# SSH 方式克隆（需要配置 SSH）
git clone git@github.com:username/repo-name.git
```

### clone 后发生了什么

```bash
git clone https://github.com/username/repo.git
cd repo

# 1. 本地创建了 main 分支，并自动跟踪 origin/main
# 2. 所有远程分支都被命名为 origin/xxx
# 3. 工作区是 main 分支的最新代码
```

**⚠️ 注意：** clone 只会拉取 main 分支（默认分支），其他分支需要之后手动切换。

### 查看远程分支

```bash
git branch -a
# 输出：
# * main
#   remotes/origin/HEAD -> origin/main
#   remotes/origin/main
#   remotes/origin/develop
#   remotes/origin/feature/payment
```

## git remote

### 查看远程仓库

```bash
git remote -v
# 输出：
# origin  git@github.com:username/repo.git (fetch)
# origin  git@github.com:username/repo.git (push)
```

### 添加远程仓库

```bash
# 添加第二个远程仓库（常用于 Fork 工作流）
git remote add upstream https://github.com/original-owner/repo.git
git remote -v
# 输出：
# origin    git@github.com:your-username/repo.git
# upstream  git@github.com:original-owner/repo.git
```

### 修改远程仓库 URL

```bash
# HTTPS 改 SSH
git remote set-url origin git@github.com:username/repo.git

# SSH 改 HTTPS
git remote set-url origin https://github.com/username/repo.git
```

## git fetch

### 基本用法

```bash
# 拉取所有远程分支的元数据
git fetch origin

# 拉取特定分支
git fetch origin feature/payment

# 拉取所有远程（所有 remote）
git fetch --all
```

### fetch 做了什么

- **只更新 refs**（refs/heads/main, refs/heads/develop 等）
- **不修改工作区文件**
- **不产生任何合并操作**

### 查看 fetch 后的差异

```bash
git fetch origin
git log main..origin/main --oneline    # 查看本地 main 领先远程多少
git diff main..origin/main            # 对比差异
```

**使用场景：** 你想知道远程有什么更新，但不确定是否要合并。用 fetch 先看看。

## git pull

### 基本用法

```bash
git pull origin main
```

### pull 的本质

`git pull = git fetch + git merge`

```bash
git pull origin main
# 等价于：
git fetch origin main
git merge origin/main
```

**⚠️ 问题：** 会产生 merge commit，导致历史分叉

### pull --rebase

```bash
git pull --rebase origin main
```

`git pull --rebase = git fetch + git rebase`

- 保持线性历史
- 不产生 merge commit
- 但会改变提交 hash

**使用场景：** 同步主分支时，优先用这个。

### 什么时候用 pull，什么时候用 fetch

| 场景 | 推荐 |
|------|------|
| 你确定要合并远程更新 | `git pull --rebase origin main` |
| 你只想看看远程有什么 | `git fetch` |
| 多人协作的公共分支 | `git pull`（merge，保留真实历史） |
| 个人功能分支同步主分支 | `git pull --rebase` |

## git push

### 基本用法

```bash
# 推送到远程
git push origin main

# 首次推送，设置上游分支
git push -u origin feature/payment

# 后续推送（设置过 -u 后）
git push
git push
```

### push 失败的原因

**场景 1：远程有你没有的提交**

```bash
# 远程有新提交，你需要先拉取
git pull --rebase origin main
git push
```

**场景 2：分支不存在**

```bash
# 首次推送创建远程分支
git push -u origin feature/payment
```

### git push --force-with-lease（安全强制推送）

```bash
git push --force-with-lease origin feature/payment
```

**为什么不用 --force：**
- `--force` 会无条件覆盖远程分支
- 如果远程有你不知道的新提交，会被覆盖

**--force-with-lease 的机制：**
- Git 检查远程最新提交是否和你本地的"预期"一致
- 如果有人在你之前推送了新提交，Git 会拒绝强制推送
- 只有确认安全的情况下才会推送

**⚠️ 绝对不要对公共分支（main/develop）使用 force push！**

## 完整远程协作流程

```bash
# 场景：你在 feature 分支开发，需要和远程同步

# 1. 确保当前分支干净
git status

# 2. 暂存当前工作（如果有未提交的修改）
git stash -m "wip: feature/payment"

# 3. 切到 main，拉取最新代码
git switch main
git pull --rebase origin main

# 4. 切回功能分支，变基到最新 main
git switch feature/payment
git rebase main

# 5. 恢复暂存的工作
git stash pop

# 6. 继续开发，commit
git add -p
git commit -m "feat(payment): complete payment integration"

# 7. 推送
git push origin feature/payment
```

## 常见问题

**Q：git pull 和 git fetch 的区别？**
A：`fetch` 只拉取元数据（refs），不合并；`pull` 是 `fetch + merge`，会直接合并。

**Q：为什么 push 的时候要输入密码/Token？**
A：说明你用的是 HTTPS 方式。建议改用 SSH，一劳永逸。

**Q：rebase 和 merge 有什么区别？**
A：见第 9 节。简单说：merge 保留分叉历史，rebase 保持线性但改变 hash。

**Q：可以同时 push 到多个远程吗？**
A：可以，通过添加多个 remote（origin、upstream 等），然后分别 push。

## 本节总结

```
克隆：
  git clone <url>                      # 克隆整个仓库
  git clone -b <branch> <url>         # 克隆指定分支

远程：
  git remote -v                       # 查看远程仓库
  git remote add upstream <url>       # 添加上游仓库

拉取：
  git fetch origin                    # 拉取元数据（不合并）
  git pull origin main                # 拉取并 merge
  git pull --rebase origin main       # 拉取并 rebase（推荐）

推送：
  git push -u origin feature/xxx      # 首次推送，设置上游
  git push                           # 后续推送
  git push --force-with-lease        # 安全强制推送
```
