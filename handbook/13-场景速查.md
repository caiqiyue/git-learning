# 13. 典型工作场景速查

## 本节目标

这不是一个"从头学习"的章节，而是一个**速查手册**。当你在工作中遇到具体问题时，来这里找对应的场景和解决方案。

---

## 场景 1：开发到一半，紧急插入 bug 修复

**问题：** 正在 `feature/payment` 分支写代码，改了 60%，突然要修线上 bug

```bash
# 1. 暂存当前工作
git add -A
git stash -m "wip: feature/payment - 60% done"

# 2. 切到 main 创建 hotfix
git switch main
git switch -c hotfix/urgent-bug

# 3. 修 bug，提交
git add -p
git commit -m "fix: resolve critical payment null pointer"
git push -u origin hotfix/urgent-bug

# 4. PR 合并后，恢复工作
git switch feature/payment
git stash pop
# 可能需要解决冲突，然后继续开发
```

---

## 场景 2：需要从别人的分支摘取一个提交

**问题：** 同事在 `feature/report-export` 分支实现了一个通用工具，你想只摘取其中一部分合入你的分支

```bash
# 1. 找到要摘取的 commit hash
git log origin/feature/report-export --oneline

# 2. 切换到你的分支
git switch feature/dashboard

# 3. 摘取特定提交
git cherry-pick -x c3d4e5f    # -x 会在 footer 记录原始 commit

# 4. 如果有冲突
git add .
git cherry-pick --continue
```

---

## 场景 3：误删分支恢复

**问题：** 不小心删除了本地 `feature/invoice生成` 分支，但远程还在

```bash
# 1. 立即用 reflog 找到分支的最后一次 commit
git reflog
# 输出示例：
# a1b2c3d HEAD@{0}: checkout: deleting branch feature/invoice生成 (was b2c3d4e)

# 2. 从那个 commit 重建分支
git switch -c feature/invoice生成 b2c3d4e

# 3. 推送回远程
git push -u origin feature/invoice生成
```

---

## 场景 4：rebase 过程中冲突中断

**问题：** 执行 `git rebase develop` 后遇到冲突，不知道下一步

```bash
# 1. 编辑冲突文件，解决冲突（删除 <<<<<<< ======= >>>>>>> 标记）

# 2. 标记冲突已解决
git add .

# 3. 继续 rebase
git rebase --continue

# 如果 rebase 进行不下去，想撤销：
git rebase --abort

# 想修改某个提交的消息：
git rebase -i HEAD~5
# 把要修改的 commit 前的 pick 改成 edit
git commit --amend
git rebase --continue
```

---

## 场景 5：多人同时开发，代码"分叉"了

**问题：** 你的分支和远程分支"分叉"了，`git push` 被拒绝

```bash
# 方式一：rebase 远程再推送（保持线性历史）
git stash                          # 先暂存你的修改
git fetch origin
git reset --hard origin/feature/payment
git stash pop                       # 恢复你的修改
git add -p
git commit -m "feat: complete feature"
git push --force-with-lease origin feature/payment

# 方式二：先 pull 合并，再推送
git pull --rebase origin feature/payment
# 如果有冲突，解决后 git add . && git rebase --continue
git push --force-with-lease origin feature/payment
```

---

## 场景 6：需要回退某个已经 push 的提交

**问题：** 提交 push 到远程后，发现错了，需要回退

```bash
# 方式一：revert（安全，适合团队协作）
git revert <commit-hash>           # 创建一个"撤销"的新提交
git push origin main

# 方式二：reset（危险，只适合本地未推送的提交）
git reset --hard HEAD~1            # 本地回退
git push --force-with-lease origin main   # 强制推送
```

---

## 场景 7：Fork 同步上游最新代码

**问题：** Fork 的仓库和上游（original repo）已经落后很多，需要同步

```bash
# 1. 添加上游仓库（如果还没有）
git remote add upstream https://github.com/original-owner/repo.git

# 2. 拉取上游最新代码
git fetch upstream

# 3. 合并到你的 main
git switch main
git merge upstream/main

# 4. 推送更新到你的 Fork
git push origin main
```

---

## 场景 8：只想要某个文件，不要整个提交

**问题：** 同事的提交里有一个工具类你想用，但不想合并整个提交

```bash
# 1. 查看那个提交里有什么文件
git show <commit-hash> --stat

# 2. 只拉取那个文件（从指定 commit 拉取）
git restore --source=<commit-hash> -- path/to/file.js

# 3. 然后可以只 commit 这个文件
git add -p path/to/file.js
git commit -m "feat: add utility class from colleague's work"
```

---

## 场景 9：用 bisect 二分查找问题提交

**问题：** 线上出了 bug，但不知道是哪次提交引入的

```bash
# 1. 开始二分查找
git bisect start

# 2. 标记当前版本有问题
git bisect bad

# 3. 标记一个已知正常的版本
git bisect good v1.0.0

# 4. Git 会自动 checkout 中间版本，测试后告诉 Git 结果
git bisect good    # 或 git bisect bad

# 5. 重复直到找到问题提交，Git 会告诉你第一个 bad commit

# 6. 结束 bisect
git bisect reset
```

---

## 场景 10：修改最后一次提交（还没 push）

**问题：** commit 后发现信息写错了，或者漏加了一个文件

```bash
# 修改 commit message
git commit --amend -m "新的正确信息"

# 追加漏掉的文件
git add forgotten-file.js
git commit --amend --no-edit    # 不改 message，只追加文件
```

⚠️ **如果已经 push 到远程，不要 amend！**

---

## 场景 11：清理本地多余的远程分支引用

**问题：** 远程分支已经删除了，但本地还有 `origin/xxx` 的引用

```bash
# 1. 清理已删除的远程分支引用
git fetch --prune
# 或简写
git fetch -p

# 2. 查看远程分支状态
git branch -r
```

---

## 场景 12：临时切换到别人的分支看看代码

**问题：** 同事说他有个分支有问题，你想本地看看但不想影响你的分支

```bash
# 1. fetch 最新分支列表
git fetch origin

# 2. 临时创建分支查看
git switch -c test-colleague origin/feature/other-dev

# 3. 查看代码、测试...
# ...

# 4. 查看完毕，切回原分支，删除临时分支
git switch my-feature
git branch -D test-colleague
```

## 本节总结

```
速查命令：
  git stash                    # 暂存当前工作
  git cherry-pick <hash>       # 摘取特定提交
  git reflog                   # 找回误删的分支
  git rebase --continue        # 继续 rebase
  git rebase --abort           # 放弃 rebase
  git push --force-with-lease  # 安全强制推送
  git revert <hash>            # 安全回退某个提交
  git bisect                   # 二分查找问题提交
  git fetch -p                 # 清理已删除的远程引用
```
