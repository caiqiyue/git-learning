# 8. 分支操作：switch / branch / push

## 本节目标

学完这节后，你会：
- 使用 `git switch`（新版命令）创建和切换分支
- 理解 `git branch`、`git push` 等基础分支操作
- 掌握分支的创建、切换、推送、删除全流程

---

## 基础分支操作

### 查看分支

```bash
git branch              # 查看本地分支（当前分支前面有 *）
git branch -a           # 查看所有分支（本地 + 远程）
git branch -r           # 只看远程分支
```

### 创建分支

```bash
# 基于当前分支创建新分支（不切换）
git branch feature/payment
```

**⚠️ 注意：** `git branch` 只是创建分支，不会切换过去。

### 切换分支

**新版命令（Git 2.23+，优先使用）**

```bash
git switch main              # 切换到已存在的 main 分支
git switch -c feature/login  # 创建并切换到新分支（新命令）
```

**旧版命令（仍可用，但不推荐）**

```bash
git checkout main            # 旧命令
git checkout -b feature/login  # 旧命令
```

### 切换到上一个分支

```bash
git switch -    # 切换到上一次所在的分支，非常实用！
```

### 基于远程分支创建本地分支

```bash
# 远程有一个 feature/login 分支，你想在本地基于它开发
git switch -c feature/login origin/feature/login
# 或者
git switch -t origin/feature/login
```

### 删除分支

```bash
git branch -d feature/old    # 删除已合并的分支
git branch -D feature/old    # 强制删除分支（即使未合并）
```

## push 到远程

### 首次推送（设置上游分支）

```bash
git push -u origin feature/payment
# -u = --set-upstream，设置上游分支
# 以后在这个分支上，git push 和 git pull 可以直接用，不用加 origin 和分支名
```

### 后续推送

```bash
git push origin feature/payment
# 或者（设置过上游分支后）
git push
```

### 安全强制推送

```bash
git push --force-with-lease origin feature/payment
```

**⚠️ --force-with-lease 的作用：**
- 普通 `git push --force` 会无条件覆盖远程分支
- `--force-with-lease` 会先检查远程是否有其他人的新提交
- 如果有人在你不知情的情况下推送了新代码，Git 会拒绝强制推送，避免覆盖别人的工作
- **团队协作中，优先使用 --force-with-lease**

### 推送所有分支

```bash
git push --all origin
```

## fetch 与 pull

### git fetch：仅拉取元数据，不合并

```bash
git fetch origin               # 拉取所有远程分支的元数据
git fetch origin feature/login  # 拉取特定远程分支
```

**特点：**
- 只更新远程指针（origin/main 等），不修改本地文件
- 不影响你的工作区
- 安全操作，不会冲突

### git pull：拉取并合并

```bash
git pull origin main
```

**等价于：**

```bash
git fetch origin main
git merge origin/main
```

**⚠️ 会产生合并提交（merge commit），导致历史分叉**

### git pull --rebase：拉取并变基

```bash
git pull --rebase origin main
```

**等价于：**

```bash
git fetch origin main
git rebase origin/main
```

**特点：**
- 不产生 merge commit，保持线性历史
- 但会"重写"你的提交（改变 commit hash）

**选择原则：**
- 同步远程主分支（develop/main）：优先用 `pull --rebase`
- 公共分支多人协作：用 `pull`（merge），保留真实历史
- 个人功能分支：习惯性用 `pull --rebase`

## fetch + log 查看远程有什么新提交

```bash
# 查看远程分支相比本地领先多少
git fetch origin
git log main..origin/main --oneline

# 对比本地和远程的差异
git diff main..origin/main
```

## 完整分支操作流程

```bash
# 场景：你从 develop 创建一个新功能分支，开发后推送到远程

# 1. 确保在 develop 分支，并拉取最新
git switch develop
git pull --rebase origin develop

# 2. 创建功能分支
git switch -c feature/payment

# 3. 开发代码...
echo "payment logic" > payment.js
git add .
git commit -m "feat(payment): initial payment module"

# 4. 推送（首次）
git push -u origin feature/payment

# 5. 开发完，PR 合并后，删除本地分支
git switch develop
git branch -d feature/payment
```

## 常见问题

**Q：git switch 和 git checkout 有什么区别？**
A：`git checkout` 在旧版 Git 中兼职太多（切换分支 + 恢复文件），容易混淆。`git switch` 专门用于切换分支，`git restore` 专门用于恢复文件，职责清晰。

**Q：-u 和 --set-upstream 是什么？**
A：设置上游分支。设置后，当前分支和远程分支建立"跟踪关系"，后续 `git push` 和 `git pull` 可以不加参数。

**Q：本地分支和远程分支一定要同名吗？**
A：不一定，但建议同名，方便管理。如果本地 `feature/login` 对应远程 `feature/login-dev`，需要显式指定。

**Q：删除远程分支怎么操作？**
A：
```bash
# 删除远程分支
git push origin --delete feature/old
```

**Q：为什么 push 到远程的分支，在 GitHub 上看不到？**
A：可能需要刷新页面。另外确认你已经推送了：`git push -u origin feature/payment`

## 本节总结

```
查看：
  git branch              # 本地分支
  git branch -a           # 所有分支
  git branch -r           # 远程分支

创建和切换：
  git switch -c feature/xxx  # 创建并切换（新命令）
  git switch main            # 切换到已有分支（新命令）
  git switch -              # 切换到上一个分支

推送：
  git push -u origin feature/xxx   # 首次推送，设置上游
  git push                           # 后续推送
  git push --force-with-lease       # 安全强制推送

拉取：
  git fetch origin              # 仅拉取元数据
  git pull origin main          # 拉取并 merge
  git pull --rebase origin main  # 拉取并 rebase（保持线性历史）

删除：
  git branch -d feature/xxx     # 删除已合并分支
  git branch -D feature/xxx     # 强制删除分支
  git push origin --delete xxx  # 删除远程分支
```
