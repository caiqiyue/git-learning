## 什么是 Pull Request

**Pull Request（PR）= 请求把你的代码拉取到主分支**

```
你：   feature/payment 分支写代码
      ↓ 创建 PR 请求
平台： Reviewer 审查代码
      ↓ 审查通过，点击 Merge
主分支：代码合并进去
```

**为什么需要 PR：**
1. 代码必须经过审查才能进入主分支
2. 所有改动都有记录，可以追溯
3. 团队成员可以评论、讨论
4. CI/CD 可以在合并前运行测试

## 创建 PR 的完整流程

### Step 1：在本地创建功能分支并开发

```bash
# 1. 确保 main 最新
git switch main
git pull --rebase origin main

# 2. 创建功能分支
git switch -c feature/payment

# 3. 开发代码...
echo "payment" > payment.js
git add .
git commit -m "feat(payment): add payment module"

# 4. 推送（首次）
git push -u origin feature/payment
```

### Step 2：在 GitHub 上创建 PR

**方式一：网页操作**

1. 推送后，GitHub 会显示提示：
   ```
   https://github.com/username/repo/tree/feature/payment
   Create Pull Request ← 点击这个链接
   ```
2. 填写 PR 信息：
   - **Title**：简短描述这次改动
   - **Description**：详细说明改动内容、动机、测试方法
   - **Reviewers**：指定审查者（可选项）
   - **Labels**：添加标签（可选）

**方式二：GitHub CLI（推荐）**

```bash
# 安装（如果没有）
brew install gh    # macOS
# Windows: 下载安装包

# 创建 PR
gh pr create \
  --title "feat(payment): add payment module" \
  --body "## Summary
- Add payment module
- Support credit card and WeChat pay

## Test plan
- [ ] Unit tests
- [ ] Manual testing" \
  --base main \
  --head feature/payment
```

### Step 3：等待 Code Review

创建 PR 后，Reviewer 会收到通知。他们会：

- 查看代码改动
- 提出评论或建议
- **Approve**（通过）或 **Request Changes**（需要修改）

---

## Code Review 操作

### 作为 Reviewer

```bash
# 1. 查看 PR 内容
gh pr view 123 --web    # 在浏览器打开
gh pr diff 123          # 查看代码差异

# 2. 拉取 PR 分支到本地查看
git fetch origin
git switch -t origin/feature/payment

# 3. 测试代码
# ... 测试 ...

# 4. 提交 Review
gh pr review 123 --approve    # 通过
# 或
gh pr review 123 --request-changes --body "需要修改以下几点..."  # 拒绝
```

### 作为 Author（作者）

收到 Review 意见后：

```bash
# 1. 查看意见
gh pr view 123

# 2. 修改代码
# ... 根据意见修改 ...

# 3. 提交并推送
git add .
git commit -m "fix: address review comments"
git push origin feature/payment

# PR 会自动更新，不需要重新创建
```

**⚠️ 注意：**
- 每次 push 后，GitHub 会通知 Reviewer
- 如果有很多小改动，建议用 `git rebase -i` 合并成有意义的提交

## PR 合并

### 合并方式

在 GitHub PR 页面，点击 **Merge** 按钮，有三种合并方式：

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| **Create a merge commit** | 保留所有分支提交历史 | 团队协作，需要追溯历史 |
| **Squash and merge** | 压缩成一个提交 | feature 分支有很多琐碎 commit |
| **Rebase and merge** | 线性历史，一个一个提交 | 保持干净历史 |

**企业推荐：** Squash and merge（保持 main 分支干净）

### 合并后操作

```bash
# 1. 本地切到 main
git switch main

# 2. 拉取最新代码（PR 已合并）
git pull --rebase origin main

# 3. 删除功能分支
git branch -d feature/payment

# 4. 删除远程分支（GitHub 会自动删除，但本地可能还有）
git push origin --delete feature/payment
```

## 完整 PR 工作流示例

```bash
# ============ 开发阶段 ============

# 1. 开始新任务，从 main 创建分支
git switch main
git pull --rebase origin main
git switch -c feature/user-auth

# 2. 开发代码
echo "user auth" > auth.js
git add -p                      # 交互式暂存
git commit -m "feat(auth): add user authentication"

# 3. 推送，创建 PR
git push -u origin feature/user-auth

# ============ Code Review 阶段 ============

# 假设 Reviewer 要求修改...

# 4. 修改代码
echo "more auth" >> auth.js
git add .
git commit --amend              # 修改最后一次提交（尚未 push）
git push --force-with-lease origin feature/user-auth

# ============ 合并后清理阶段 ============

# 5. PR 合并后
git switch main
git pull --rebase origin main

# 6. 清理分支
git branch -d feature/user-auth

# 7. 开始下一个任务
git switch -c feature/new-task
```

## Fork 工作流（外部贡献）

如果没有仓库的写权限，需要用 Fork：

```bash
# 1. 在 GitHub 上 Fork 仓库

# 2. 克隆你 Fork 的仓库
git clone https://github.com/your-username/repo.git
cd repo

# 3. 添加上游仓库
git remote add upstream https://github.com/original-owner/repo.git

# 4. 创建分支开发
git switch -c feature/new-feature
git add -p
git commit -m "feat: add new feature"
git push origin feature/new-feature

# 5. 在 GitHub 上创建 PR（base 是 original-owner/repo）
# 注意：head 是 your-username/repo

# 6. PR 合并后，同步上游
git fetch upstream
git switch main
git merge upstream/main
git push origin main

# 7. 删除功能分支
git branch -d feature/new-feature
```

## 常见问题

**Q：PR 可以创建到非 main 分支吗？**
A：可以。可以创建到任何分支，比如 `develop`、`release/1.0` 等。

**Q：可以给同一个 PR 多次提交吗？**
A：可以。在 PR 合并前，你可以 push 任意次。每次 push 都会更新 PR。

**Q：Request Changes 后怎么办？**
A：根据意见修改代码，push 后 PR 会自动更新。Reviewer 会重新审查。

**Q：PR 和 MR 有什么区别？**
A：GitHub 叫 Pull Request（PR），GitLab 叫 Merge Request（MR）。功能一样。

**Q：可以取消已创建的 PR 吗？**
A：可以。在 GitHub PR 页面点击 "Close pull request"。关闭后如果需要可以重新打开。

## 本节总结

```
创建 PR 流程：
  1. git switch main && git pull --rebase origin main
  2. git switch -c feature/xxx
  3. 开发，commit
  4. git push -u origin feature/xxx
  5. GitHub 上创建 PR

Code Review：
  - Reviewer 查看代码，提出意见
  - Author 修改后 push --force-with-lease

合并：
  - Squash and merge（推荐）
  - 或 Create a merge commit

合并后：
  git switch main && git pull --rebase origin main
  git branch -d feature/xxx

Fork 工作流：
  git clone your-fork.git
  git remote add upstream original-repo.git
  # 开发后创建 PR 到 original-repo
  git fetch upstream && git merge upstream/main
```
