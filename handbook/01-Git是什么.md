# 1. Git 是什么？为什么要用？

## 本节目标

学完这节后，你会明白：
- Git 是什么，它解决什么问题
- 为什么不直接把代码发给同事，而是要用 Git
- Git 和 GitHub / GitLab 是什么关系

---

## Git 是什么

**Git 是一个版本控制系统**（Version Control System），它的核心能力是：**记录你代码的每一次变更，并且可以随时回到任何一个历史版本**。

就像游戏的**存档功能**——你打 Boss 前存个档，打输了就读档重来。代码写崩了？Git 帮你"读档"到上一个正确的版本。

---

## 为什么要用 Git（而不是直接发文件）

**场景还原：**

小明和小王合作开发一个项目，没有 Git。

```
第1天：小明把代码发微信 → 小王在代码基础上改 → 再发回
第3天：小王说"我改了你那个文件，但忘了改哪个了"
第5天：线上出 bug，想回退，但不知道哪个版本是好的
```

这就是不用版本控制的痛苦——**代码历史一团浆糊，协作全靠吼**。

**有了 Git 之后：**

```
小明：git commit "新增用户登录功能"
小明：git push origin main        # 推送到 GitHub

小王：git pull origin main        # 拉取最新代码
小王：git switch -c feature/payment  # 创建自己的分支
小王：改代码，commit，推送，创建 PR

小明：Code Review 通过 → Merge
```

每个人都在自己的分支上干活，互不干扰。代码经过审查才能合并到主分支。**每一次变更都有记录，谁改了什么都清清楚楚**。

---

## Git 解决的核心问题

| 问题 | Git 的解决方案 |
|------|--------------|
| 代码版本混乱 | 每次提交都有记录，可随时回退 |
| 多人协作冲突 | 分支隔离 + 合并机制 |
| 误删代码找不回 | 完整的操作历史（reflog） |
| 不知道谁改了什么 | 每行代码可追溯（blame） |
| 线上出 bug 定位不到 | bisect 二分查找定位问题提交 |

## Git 和 GitHub / GitLab 是什么关系

很多人搞混这三个概念：

```
Git     = 本地软件（安装在你电脑上的版本控制工具）
GitHub  = 线上平台（托管代码的网站，你把本地 Git 的仓库上传到这里）
GitLab  = 另一个线上平台（功能类似 GitHub，企业自建用这个）
Gitee   = 国内的类似平台
```

类比理解：

```
Git     = 相机（拍照的工具）
GitHub  = 云相册（存放照片的地方）

没有 GitHub，你的照片只在你手机上
有了 GitHub，别人能看到、评论、下载你的照片
```

**一个重要澄清：** Git 可以不连 GitHub 用（纯本地版本控制），但加上 GitHub 之后，就有了"备份 + 协作"的能力。本手册重点讲本地 Git 操作 + GitHub 协作。

---

## Git 的工作方式：快照而非差异

这是理解 Git 的关键。

**传统版本控制**（如 SVN）存储的是"每次改了什么"（差异）：

```
v1: 初始代码
v2: 修改了第 5 行
v3: 修改了第 8 行
```

**Git** 存储的是"每个版本的完整快照"：

```
v1: [完整代码快照 A]
v2: [完整代码快照 B]  ← Git 实际存储的是快照
v3: [完整代码快照 C]     但通过"内容寻址"机制，
                        相同文件只存一份，不占空间
```

这让 Git 的分支极其"便宜"——创建分支只是创建一个指针，不复制任何文件。

## 常见问题

**Q：Git 和 GitHub 是同一个东西吗？**
A：不是。Git 是工具，GitHub 是平台。就像"微信"和"手机"的关系——微信是软件，GitHub 是托管这个软件数据的网站。

**Q：我一定要用 GitHub 吗？**
A：不一定。GitLab、Gitea、码云都可以。但 GitHub 是最大的平台，本手册以它为例。

**Q：Git 难学吗？**
A：基础操作（add / commit / push / pull）10 分钟就能上手。高级操作需要多练。这本手册会带你从零到实战。

## 扩展了解

- **Git 诞生背景**：2005 年，Linus Torvalds（Linux 之父）为了管理 Linux 内核代码，开发了 Git。它是免费的开源工具。
- **Git 的特点**：分布式（每个开发者本地都有完整仓库）、性能极快、安全（SHA-1 哈希校验）
- **Git 的竞争对手**：Mercurial、Subversion（SVN）等。但 Git 是目前最主流的，几乎是行业标准。
