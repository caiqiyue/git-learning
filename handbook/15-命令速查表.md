# 15. 命令速查表

## 本节目标

这是手册的**最后一页**，一个按场景分类的速查表。你不需要记住所有命令，只需要知道**遇到问题时去哪里查**。

---

## 日常操作（每天用）

```bash
git switch -c <branch>              # 创建并切换分支（新命令）
git switch <branch>                 # 切换分支
git switch -                        # 切换到上一个分支

git add <file>                      # 暂存单个文件
git add .                           # 暂存所有变更
git add -p                          # 交互式暂存（逐块选择）⭐

git commit -m "type(scope): msg"   # 提交
git commit --amend                 # 修改最后一次提交（未 push 时）

git status                          # 查看状态
git log --oneline                   # 查看最近提交（单行）
git log --oneline --graph           # 图形化提交历史
```

---

## 分支操作

```bash
git branch                          # 查看本地分支
git branch -a                       # 查看所有分支
git branch -r                       # 只看远程分支

git switch -c feature/xxx           # 创建并切换
git switch -t origin/feature/xxx    # 基于远程分支创建

git push -u origin <branch>         # 首次推送，设置上游
git push                           # 后续推送
git push --force-with-lease        # 安全强制推送

git branch -d <branch>              # 删除已合并分支
git branch -D <branch>              # 强制删除分支
git push origin --delete <branch>   # 删除远程分支
```

---

## 远程操作

```bash
git clone <url>                     # 克隆仓库
git remote -v                       # 查看远程仓库
git remote add upstream <url>       # 添加上游仓库

git fetch origin                    # 拉取元数据（不合并）
git fetch --all                     # 拉取所有远程
git fetch -p                        # 清理已删除的远程引用

git pull origin main                # 拉取并 merge
git pull --rebase origin main       # 拉取并 rebase（保持线性历史）

git push                            # 推送
```

---

## 撤销与回退

```bash
git restore <file>                  # 丢弃工作区修改（新命令）
git restore --staged <file>         # 取消暂存（新命令）
git restore .                       # 丢弃所有工作区修改

git reset --soft HEAD~1             # 回退 1 个提交，保留暂存区
git reset HEAD~1                    # 回退 1 个提交，取消暂存
git reset --hard HEAD~1             # 回退 1 个提交，丢弃一切（危险！）

git reset --hard <commit-hash>      # 回退到指定版本
git revert <commit-hash>           # 撤销指定提交（安全，适合团队）
```

---

## stash 暂存

```bash
git stash -m "描述"                 # 暂存当前工作
git stash -u                         # 暂存包括未跟踪文件
git stash list                       # 查看所有 stash
git stash pop                        # 恢复并删除最近 stash
git stash apply                      # 恢复但不删除
git stash drop stash@{0}            # 删除指定 stash
git stash clear                      # 清空所有 stash
```

---

## 分支合并与冲突

```bash
git merge feature/xxx               # 合并分支
git rebase main                      # 变基到 main（重写 hash！）

# 冲突解决
git add <resolved-file>              # 标记冲突已解决
git commit                           # 完成 merge
git rebase --continue                # 继续 rebase
git merge --abort                    # 放弃 merge
git rebase --abort                   # 放弃 rebase

# 合并策略
git merge --squash feature/xxx       # 压缩合并（多个提交变 1 个）
```

## 查询与追溯

```bash
git log --oneline -10                # 最近 10 条提交
git log --oneline --graph            # 图形历史
git log --oneline --all              # 所有分支历史
git log --author="name"             # 按作者筛选
git log --since="2024-01-01"        # 按时间筛选
git log -S "string"                  # 搜索提交内容

git show <commit-hash>              # 查看某个提交详情
git diff main..feature/xxx          # 对比两个分支差异
git diff --name-only                # 只看改动的文件名

git blame <file>                    # 逐行追溯责任人
git shortlog -sn                    # 贡献者统计
```

## cherry-pick 与 bisect

```bash
git cherry-pick <hash>              # 摘取单个提交
git cherry-pick -x <hash>           # 摘取并记录原始 commit
git cherry-pick <h1> <h2> <h3>      # 摘取多个提交
git cherry-pick --no-commit <hash>  # 摘取但不自动提交
git cherry-pick --continue          # 继续摘取（冲突后）
git cherry-pick --abort             # 放弃摘取

git bisect start                     # 开始二分查找
git bisect bad                       # 标记当前版本有问题
git bisect good v1.0.0               # 标记已知正常版本
git bisect good / git bisect bad     # 重复测试
git bisect reset                     # 结束二分查找
```

## reflog 救命机制

```bash
git reflog                           # 查看所有操作历史
git reflog | head -20                # 查看最近 20 条

# 恢复误删的分支
git reflog
# 找到分支最后一次 commit 的 hash，如 abc123
git switch -c feature/restored abc123

# 恢复误 reset 的提交
git reflog
git reset --hard abc123
```

## SSH 与 GitHub

```bash
ssh-keygen -t ed25519 -C "邮箱"     # 生成 SSH 密钥
cat ~/.ssh/id_ed25519.pub            # 查看公钥
ssh -T git@github.com               # 验证 SSH 连接

git remote set-url origin git@github.com:username/repo.git  # 改用 SSH
```

## 其他常用命令

```bash
git init                             # 初始化新仓库
git tag -a v1.0.0 -m "release"       # 创建标签
git tag                              # 查看标签
git push origin --tags              # 推送所有标签

git config --list                   # 查看配置
git config --global user.name "名字"  # 设置用户名
git config --global user.email "邮箱" # 设置邮箱

# 命令别名（放在 ~/.gitconfig 的 [alias] 下）
git config --global alias.st status
git config --global alias.lg "log --oneline --graph --all"
```

## 命令速查索引

| 问题 | 命令 |
|------|------|
| 怎么创建分支？ | `git switch -c feature/xxx` |
| 怎么暂存代码？ | `git add -p`（逐块）或 `git add .`（全部） |
| 怎么提交？ | `git commit -m "type(scope): msg"` |
| 怎么推送到远程？ | `git push -u origin <branch>` |
| 怎么拉取最新代码？ | `git pull --rebase origin main` |
| 怎么撤销工作区修改？ | `git restore <file>` |
| 怎么取消暂存？ | `git restore --staged <file>` |
| 怎么回退版本？ | `git reset --hard <commit>` |
| 怎么临时保存工作？ | `git stash -m "描述"` |
| 怎么恢复 stash？ | `git stash pop` |
| 怎么解决冲突？ | 编辑文件 → `git add .` → `git commit` |
| 怎么从别人分支摘取代码？ | `git cherry-pick <hash>` |
| 怎么恢复误删的分支？ | `git reflog` + `git switch -c <branch> <hash>` |
| 怎么强制推送？ | `git push --force-with-lease`（安全） |
