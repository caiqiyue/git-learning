# 3. 安装与初始配置

## 本节目标

学完这节后，你会：
- 在 Windows / macOS / Linux 上正确安装 Git
- 配置好用户信息（必须的第一步）
- 了解 Git 的基本配置层级

---

## 安装 Git

### Windows

**方式一：Git Bash（推荐）**

1. 打开 https://git-scm.com/download/win
2. 下载安装包，一路下一步即可
3. 安装完成后，右键任意文件夹 → "Git Bash Here" 打开终端

**方式二：WSL（Windows Subsystem for Linux）**

如果你使用 WSL，Git 通常已经预装。验证方式：

```bash
git --version
# 输出：git version 2.x.x
```

如果没有：

```bash
sudo apt update
sudo apt install git
```

### macOS

**方式一：Homebrew（推荐）**

```bash
# 如果没有 Homebrew，先安装：
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装 Git
brew install git
```

**方式二：Xcode Command Line Tools**

```bash
git --version
# 如果提示未安装，执行：
xcode-select --install
```

### Linux

```bash
# Debian/Ubuntu
sudo apt update
sudo apt install git

# Fedora
sudo dnf install git

# 验证安装
git --version
```

## 首次配置（必须！）

安装完 Git 后，**第一件事**是配置你的身份信息。Git 会把这个信息写入每一次提交。

### 配置用户名和邮箱

```bash
# 全局配置（所有项目都用这个身份）
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

**⚠️ 注意：**
- 邮箱建议使用和 GitHub 账号相同的邮箱，后续操作更方便
- `--global` 表示全局配置，如果不加 `--global`，只对当前仓库生效

### 验证配置

```bash
git config --list
# 输出：
# user.name=你的名字
# user.email=你的邮箱@example.com
```

### 为什么要配置这个

每次你 `git commit`，Git 会把你的用户名和邮箱写入提交记录。这个信息是公开的（通过 GitHub 查看），所以：
- 使用真实姓名（方便同事认识你）
- 使用常用邮箱

## 初始化一个新仓库

有两种方式开始使用 Git：

### 方式一：从头创建（项目开始时）

```bash
# 进入项目文件夹
cd ~/my-project

# 初始化仓库（在文件夹里创建 .git 目录）
git init
# 输出：Initialized empty Git repository in /path/to/my-project/.git/
```

### 方式二：克隆已有仓库（加入项目时）

```bash
# 克隆远程仓库到本地
git clone https://github.com/username/repo-name.git

# 如果想指定文件夹名称：
git clone https://github.com/username/repo-name.git my-folder-name
```

## Git 配置的三种层级

Git 配置有三个层级，从高到低：

| 层级 | 作用域 | 配置方式 |
|------|--------|---------|
| `--system` | 当前操作系统所有用户 | 很少用，需要管理员权限 |
| `--global` | 当前用户的所有仓库 | **最常用** |
| `--local` | 当前仓库 | 只对当前仓库生效 |

优先级：**local > global > system**

### 查看各级配置

```bash
git config --list --local      # 当前仓库配置
git config --list --global     # 当前用户全局配置
git config --list --system     # 系统配置
```

### 实际场景

```bash
# 公司项目用公司邮箱，个人项目用私人邮箱
# 在公司项目目录下：
git config --local user.email "zhangsan@company.com"

# 全局还是私人邮箱
git config --global user.email "my-private@email.com"
```

## 推荐的额外配置

### 1. 设置默认分支名为 main（而非 master）

新版本 Git 默认分支已经是 main，但建议显式设置：

```bash
git config --global init.defaultBranch main
```

### 2. 设置默认编辑器

如果你习惯用 VSCode：

```bash
git config --global core.editor "code --wait"
```

### 3. 设置别名（可选）

```bash
# 给常用命令设置短别名
git config --global alias.st status
git config --global alias.lg "log --oneline --graph --all"
```

以后输入 `git st` 就等于 `git status`。

### 4. 彩色输出

```bash
git config --global color.ui auto
```

## 常见问题

**Q：提示 "Please tell me who you are" 怎么解决？**
A：这是因为你没有配置用户名和邮箱。执行：
```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

**Q：--global 和 --local 有什么区别？**
A：`--global` 是当前用户的全局配置，所有项目都生效；`--local` 只对当前仓库生效。

**Q：可以修改已配置的信息吗？**
A：可以，直接重新执行配置命令即可覆盖。

**Q：配置信息存在哪里？**
A：
```
--local:   .git/config（当前仓库）
--global:  ~/.gitconfig（用户目录）
--system:  /etc/gitconfig（系统目录）
```
